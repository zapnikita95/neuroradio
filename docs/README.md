# Music Story — инструкция по запуску и тестированию

Приложение **Music Story** для Android перехватывает метаданные текущего трека из **Spotify** и **Яндекс Музыки**, запрашивает короткую «радио-историю» у бэкенда и озвучивает её поверх музыки (или через локальный TTS, если аудио с сервера недоступно).

## Структура проекта

```
Music story/
├── android/          # Android-приложение (APK)
├── backend/          # Node.js BFF (истории + Yandex TTS)
└── docs/             # Документация
```

## Требования

- **Android**: устройство или эмулятор с API 26+, Spotify и/или Яндекс Музыка
- **Backend**: Node.js 18+
- **Сеть**: телефон и компьютер в одной Wi‑Fi сети (для теста на реальном устройстве)
- **Опционально**: ключи Groq и Yandex SpeechKit для полноценных историй и серверного аудио

## 1. Запуск бэкенда

```bash
cd backend
npm install
cp .env.example .env
npm run dev
```

Сервер слушает `http://localhost:3000`.

### Demo-режим (без API-ключей)

Если в `.env` нет `GROQ_API_KEY` или ключей Yandex, бэкенд работает в **demo-режиме**:

- возвращает готовый русский текст в поле `script`;
- поле `audioUrl` = `null`, `demo` = `true`;
- Android озвучивает историю **системным TextToSpeech**.

### Полный режим

Добавьте в `.env`:

| Переменная | Назначение |
|------------|------------|
| `GROQ_API_KEY` | Генерация текста истории |
| `YANDEX_API_KEY` | Yandex SpeechKit |
| `YANDEX_FOLDER_ID` | ID каталога Yandex Cloud |

Тогда бэкенд вернёт `audioUrl` (например `/audio/<uuid>.ogg`), и приложение воспроизведёт готовое аудио.

Проверка:

```bash
curl http://localhost:3000/health
curl -X POST http://localhost:3000/v1/story/full \
  -H "Content-Type: application/json" \
  -d "{\"artist\":\"Кино\",\"title\":\"Группа крови\"}"
```

## 2. Сборка Android-приложения

```bash
cd android
./gradlew assembleDebug
```

APK: `android/app/build/outputs/apk/debug/app-debug.apk`

Установка на подключённое устройство:

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

## 3. URL бэкенда в приложении

Откройте **Настройки → URL бэкенда**:

| Где тестируете | URL |
|----------------|-----|
| Android Emulator | `http://10.0.2.2:3000` |
| Реальный телефон | `http://192.168.x.x:3000` (IP вашего ПК в локальной сети) |

Как узнать IP на Windows:

```powershell
ipconfig
```

Ищите **IPv4-адрес** Wi‑Fi адаптера, например `192.168.1.42`. Тогда URL: `http://192.168.1.42:3000`.

> Убедитесь, что брандмауэр Windows разрешает входящие подключения на порт 3000.

## 4. Тестирование со Spotify

1. Установите [Spotify](https://play.google.com/store/apps/details?id=com.spotify.music).
2. Установите и откройте **Music Story**.
3. На экране онбординга выдайте **доступ к уведомлениям**:
   - Настройки Android → Приложения → Специальный доступ → **Доступ к уведомлениям**
   - Включите **Music Story**
4. Запустите воспроизведение любого трека в Spotify.
5. В Music Story на главном экране должны появиться **артист и название**.
6. Дождитесь автоматической истории (по умолчанию каждые 10 треков) или нажмите **«Рассказать историю»**.

### Что проверить

- [ ] Метаданные трека определяются
- [ ] Музыка приостанавливается на время истории
- [ ] После истории музыка возобновляется
- [ ] В demo/offline режиме слышен русский TTS

---

## 5. Тестирование с Яндекс Музыкой

1. Установите [Яндекс Музыку](https://play.google.com/store/apps/details?id=ru.yandex.music).
2. Повторите шаги 2–6 из раздела Spotify.
3. При одновременном воспроизведении в нескольких плеерах приложение отдаёт приоритет Spotify, затем Яндекс Музыке.

Поддерживаемые package name:

- `com.spotify.music`
- `ru.yandex.music`
- `com.yandex.music`

---

## 6. Режимы работы приложения

| Режим | Описание |
|-------|----------|
| **Автоматический** | История по триггеру (каждые N треков, всегда и т.д.) |
| **Ручной** | Только по кнопке «Рассказать историю» |
| **Автоперехват** | Пауза музыки и вставка истории |

Настройки триггера: **Настройки → Режим триггера**.

## 7. Работа без сети (локальный fallback)

Если бэкенд недоступен (нет Wi‑Fi, неверный URL, сервер не запущен), приложение **не падает с ошибкой**:

1. Пытается взять историю из локального кэша (24 ч).
2. Если кэша нет — генерирует короткую русскую историю через `LocalStoryGenerator` (~60 слов).
3. Озвучивает её **Android TextToSpeech** (`audioUrl` = null).

Это поведение совпадает с **demo-режимом бэкенда**, когда сервер отвечает без `audioUrl`.

## 8. Типичные проблемы

### Трек не определяется

- Проверьте доступ к уведомлениям для Music Story.
- Убедитесь, что служба мониторинга активна (уведомление «Ожидание музыки…»).
- Перезапустите плеер и приложение.

### История не загружается (ошибка сети)

- Проверьте URL бэкенда и IP компьютера.
- На эмуляторе используйте `10.0.2.2`, не `localhost`.
- Запустите `npm run dev` в папке `backend`.

### Нет звука истории

- В demo/offline режиме нужен установленный **русский голос TTS** (Настройки Android → Язык и ввод → Синтез речи).
- Проверьте громкость медиа.

### Spotify / Яндекс не в списке

Приложение читает метаданные через `NotificationListenerService` и MediaSession — оба плеера должны показывать медиа-уведомление при воспроизведении.
