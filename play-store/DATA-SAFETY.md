# Google Play — Безопасность данных (шпаргалка)

## Сбор данных

**Ваше приложение собирает данные?** → **Да**

**Передаются в зашифрованном виде?** → **Да** (весь API — HTTPS). Не ставь «Нет».

## Создание аккаунта — отметить

| Вариант | Нужно? |
|---------|--------|
| Имя пользователя и пароль | **Нет** (пароля нет) |
| **Имя пользователя и другие данные для подтверждения личности** | **Да** — вход по **email + одноразовый код** |
| **OAuth** | **Да** — вход через **Telegram** |
| Другое | по желанию |
| В приложении нельзя создать аккаунт | **Нет** — аккаунт создаётся при первом входе |

## URL удаления аккаунта

```
https://www.efir-ai.ru/docs/delete-account.html
```

(или `https://efir-ai.ru/docs/delete-account.html` — оба домена ведут на сайт)

## Типы данных (типовой набор для Эфир AI)

При детальной анкете отметь примерно:

| Тип | Собирается | Передаётся | Обязательно |
|-----|------------|------------|-------------|
| Email | Да | Нет (или только процессорам) | Нет (для входа/оплаты) |
| История действий в приложении | Да | Нет | Нет |
| Другие действия (метаданные треков) | Да | Нет | Да (основная функция) |
| Идентификаторы устройства | Да | Нет | Да |
| Покупки | Да | Платёжному партнёру | Нет |

Цели: функции приложения, аналитика/стабильность (если спросят), управление аккаунтом.

Удаление: пользователь может запросить по странице delete-account.html.
